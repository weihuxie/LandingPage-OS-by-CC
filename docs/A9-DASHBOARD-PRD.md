# A9 多产品增长看板 — PRD

> 附属于 LandingPage OS PRD v5.1 §7
> 目标读者：需要横向管理多条业务线的 CEO / CMO / Growth Lead

---

## 1. 目标与定位

一张屏看完所有业务线在做什么、跑得怎么样、下一步该动哪儿。

不是一个 BI 工具，而是一个**经营决策的驾驶舱**。看完后应该能直接：
1. 知道哪些产品在健康增长，哪些需要补资产
2. 知道每个市场（CN/TW/JP/US）的转化质量差异
3. 知道哪些 A/B 测试可以结束并固化方案
4. 拿到 AI 给出的、可执行的下一步动作清单

看板必须在 3 秒内把"值得关注的事"推到眼前，剩下才是明细。

---

## 2. 核心用户旅程

```
CEO 早上打开看板
  → 3 秒内看到「昨日总留资 / 环比 / 告警数」
  → 扫一眼「需要我动手的事」（AI 优化任务池）
  → 点击某条任务 → 直达对应项目编辑器
  → 全部处理完 → 回看板 → 关闭浏览器
```

## 3. 页面结构

### 3.1 顶部 KPI 条（4 格）
- **产品数**：所有项目 count
- **总访问 UV**：window = 默认 7 天，可切 1/7/30/90 天
- **总留资**：同周期
- **平均转化率**：Leads / UV，环比箭头

### 3.2 产品横向对比表
每一行一个产品，按 Leads 降序。列：

| 列 | 说明 |
|---|---|
| 产品 | 名称（点击进编辑器） |
| 市场 | CN / TW / JP / US / EU / GLOBAL |
| 模式 | 单方案 / A-B 分流 |
| UV | 周期内 |
| Leads | 周期内 |
| CVR | Leads / UV |
| CPL | 若接了广告 → 留资成本 |
| A/B | 胜出方案标签 / 样本不足 / 无显著差异 |
| 趋势 | 7 天 sparkline |

可按市场、发布状态、模式筛选。

### 3.3 按语言分布
横向条形图，每语言一行：UV 占比 + 该语言的 CVR。
辅助判断"是否该给某市场单独加资产"。

### 3.4 地理热力图（Phase 2）
世界地图，访客数按国家着色。按住产品筛选。
数据源 = IP → 国家（开源 MaxMind DB，每月更新）。

### 3.5 AI 优化任务池
每条任务一个卡片，优先级从高到低。内置规则（可扩展）：

| 触发条件 | 建议 |
|---|---|
| UV ≥ 300 且 CVR < 1% | 重写首屏或缩短表单 |
| JP 市场 CVR < 1.5% | 补 SOC 2 / ISMS 认证资产，切极简信任风格 |
| A/B 分流已有显著胜出（lift ≥ 10%, 样本 ≥ 200×2） | 固化为单方案 |
| 表单字段 ≥ 5 且 CVR < 平均 | 减字段 |
| 已发布 30 天但无更新 | 跑一次 AI 审计 |
| 首屏 CTA 与 Footer CTA 不一致 | 统一主 CTA |

每条任务附：**「一键跳到对应项目 / 一键让 AI 生成修复方案」**。

## 4. 数据模型

### 4.1 事件模型（最小）
```ts
PageEvent {
  id, projectId, variant?: 'A'|'B',
  type: 'view' | 'cta_click' | 'form_submit',
  locale, country?, referrer?,
  createdAt
}
```
- view：首屏进入视口
- cta_click：任一 CTA 被点击
- form_submit：表单成功提交（同 Lead）

### 4.2 聚合路径
API `GET /api/analytics`：对事件表做 aggregation。
MVP 全量扫描（项目量 < 1k），Phase 2 用 ClickHouse / DuckDB 做物化视图。

### 4.3 A/B 胜出算法
- 样本阈值：A 和 B 各 ≥ 200 UV
- 显著性：lift 绝对值 ≥ 5% 才给方向
- 置信度：Phase 2 接 Beta 分布，输出"B 胜 A 的概率 = 87%"

---

## 5. 交互细节

### 5.1 周期切换
右上角时间选择器：今日 / 7 天（默认）/ 30 天 / 90 天 / 自定义。
切换时只重算聚合，项目表不改变。

### 5.2 多产品筛选
左侧可折叠侧栏：
- [ ] One-Flow
- [ ] WinPilot
- [ ] …

多选时所有图表数据做交集合并。

### 5.3 导出
右上角「导出 CSV」按钮：把横向对比表导成 csv，方便粘到周报。

### 5.4 钻取
点击任一产品名 → 进入该产品的单独分析视图（Phase 2：漏斗 / 热图 / 回放）。

---

## 6. 权限与数据隔离

- 账号绑定：只看到自己拥有的项目（RLS 基于 user_id）
- Team 级别（Phase 2）：owner / admin / viewer 三档
- 审计日志：谁在什么时候切换了 A/B、发布了页面 — 留痕

---

## 7. 性能目标

- P95 首次加载 < 1.5s（项目数 ≤ 50、事件数 ≤ 100k）
- 切换周期 < 300ms
- 热力图首帧 < 2s

---

## 8. 阶段划分

### Phase 1 (MVP, 已实现)
- 顶部 KPI 条
- 产品横向对比表
- 按语言分布
- AI 优化任务池（6 条内置规则）
- 周期默认 7 天（无切换）

### Phase 2
- 时间周期切换
- 地理热力图
- 7 天 sparkline + CPL
- A/B 置信度（Beta 分布）
- 单产品钻取视图

### Phase 3
- Team 权限、审计日志
- 导出 CSV / PDF 周报
- Slack / 飞书 定期推送
- 异常告警（CVR 断崖、UV 暴跌）

## 10. 成功指标

- 每周至少打开 3 次的 CEO 用户占比 ≥ 60%
- 任务池建议被点击执行率 ≥ 25%
- 从看板跳转到编辑器并实际修改发布的比例 ≥ 15%
