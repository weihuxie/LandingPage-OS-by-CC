## 1. L1 纯函数补漏

### 1.1 `slug.ts` · `makeSlug(name)`

**规约**（反推自 `slug.ts:3-14`）：
- 输入 `name: string`，输出形如 `<base>-<6位随机>`
- `base` = 输入 → 小写 → NFKD 去重音 → 去非 `\w\s-` → trim → 空格转 `-` → 截断 32 字符
- 空 `name` / 纯特殊字符 → `base = ''`，则只返回 6 位 nanoid

**等价类**：
- 合法：英文 / 数字 / 含空格 / 含 hyphen
- 边界：长度 = 32 / > 32 / < 6 / 完全空
- 异常：`null`/`undefined` 走 `|| 'page'` 兜底 → `'page-xxxxxx'`
- Unicode：CJK（不在 `\w` 集，会被剥光 → 回 6 位 nanoid）；带重音的拉丁字符（NFKD 应转成 ASCII）；emoji（剥光）

**用例**：

| ID | 输入 | 期望 | 优先级 |
|---|---|---|---|
| **API-SLUG-401** | `"Hello World"` | `/^hello-world-[a-z0-9]{6}$/` | 🟡 |
| **API-SLUG-402** | `""` | `/^[a-z0-9]{6}$/`（仅后缀，无破折号前缀）| 🟡 |
| **API-SLUG-403** | `null as any` | `/^page-[a-z0-9]{6}$/`（`|| 'page'` 兜底）| 🟢 |
| **API-SLUG-404** | `"a".repeat(50)` | base 部分长度 = 32（截断） | 🟡 |
| **API-SLUG-405** | `"中文产品"` | `/^[a-z0-9]{6}$/`（CJK 全剥）| 🟡 |
| **API-SLUG-406** | `"café"` | `/^cafe-[a-z0-9]{6}$/`（NFKD 去重音） | 🟡 |
| **API-SLUG-407** | `"!!!@@@###"` | `/^[a-z0-9]{6}$/`（特殊符全剥） | 🟢 |
| **API-SLUG-408** | `"Hello   World"` | `/^hello-world-[a-z0-9]{6}$/`（多空格折叠） | 🟢 |
| **API-SLUG-409** | 1000 次调用同输入 | 全部唯一（nanoid 防撞） | 🟡 |
| **API-SLUG-410** | `"foo--bar"` | hyphen 保留，不被合并 | 🟢 |

### 1.2 `i18n-detect.ts`

**规约**（反推自 `i18n-detect.ts:19-53`）：
- `detectLocale(ctx)` 优先级：URL > Cookie > Accept-Language > Country > fallback
- 必须返回 `available` 中的 locale，否则 fallback
- `parseAcceptLanguage`：按 `q=` 排序
- `bestMatch`：`zh-CN/zh/zh-Hans/zh-SG` → `zh-CN`；`zh-TW/zh-HK/zh-MO/zh-Hant` → `zh-TW`；`en*` → `en`；`ja*` → `ja`
- `countryToLocale`：JP→ja / TW,HK,MO→zh-TW / CN,SG→zh-CN / 其他→en

**等价类**：
- 优先级：每层独立命中、上层不命中下层兜底、全部不命中 fallback
- Accept-Language：单 tag / 多 tag 带 q / q 异常 / 大小写混合 / 未注册变体
- 不在 `available`：URL/Cookie 强制传非 available → 必须降级
- Country 大小写、未列出国家

**用例**：

| ID | 输入 | 期望 | 优先级 |
|---|---|---|---|
| **API-I18N-401** | `urlLang='ja'`, available=`['zh-CN','ja']` | `{locale:'ja',source:'url'}` | 🟡 |
| **API-I18N-402** | URL='ja' 但 available=`['zh-CN','en']`，cookie='en' | `{locale:'en',source:'cookie'}`（URL 不可用降级到 cookie）| 🟡 |
| **API-I18N-403** | `acceptLanguage='zh-CN,zh;q=0.9,en;q=0.8'`, available=`['en','zh-CN']` | `{locale:'zh-CN',source:'accept-language'}` | 🟡 |
| **API-I18N-404** | `acceptLanguage='en;q=0.5,ja;q=0.9'`, available=`['en','ja']` | `{locale:'ja',source:'accept-language'}`（按 q 排序） | 🟡 |
| **API-I18N-405** | `acceptLanguage='zh-Hant;q=1'`, available=`['zh-TW']` | 命中 `zh-TW` | 🟡 |
| **API-I18N-406** | `acceptLanguage='zh-Hans'`, available=`['zh-CN']` | 命中 `zh-CN` | 🟡 |
| **API-I18N-407** | `country='JP'`, available=`['ja','en']`，无 URL/cookie/accept | `{locale:'ja',source:'country'}` | 🟡 |
| **API-I18N-408** | `country='br'`（未列国家小写）, available=`['en']` | `{locale:'en',source:'country'}`（默认 en） | 🟢 |

### 2.2 `llm-fallback.ts` · 编排器路径

**规约**（反推自 `llm-fallback.ts:72-192`）：
- `executeScenario(scenario, locale, executor)`：读 `policyFor(cfg, scenario, locale)`，按 `chain` 顺序尝试
- 每步：跳过 no-key（除 `mode==='skip-polish'`） → 调 executor → 成功返回 `{result, usedStep, hops}`
- 失败时分类：`LLMRequiredError` → 直接抛；`4xx-auth/4xx-other` → 短路（同输入换家无意义）；`!isTriggered` → 短路；否则下一步
- chain 全跑完仍失败：抛 firstError
- **2026-05 bug fix**：所有 step 都因 no-key 跳过 → `firstError===null`，必须抛 `LLMRequiredError`（不是裸 `Error`）
- `policy.enabled===false` 或 `chain.length===0` → 抛 `LLMRequiredError`
- `network` cls 视同 5xx 触发判断

**等价类**：
- 步骤数：1 / 2 / N
- 每步状态：has-key+success / has-key+fail-trigger / has-key+fail-non-trigger / no-key（skip）/ skip-polish 模式
- 失败类型：`429-quota` / `429-rate` / `5xx` / `4xx-auth` / `4xx-other` / `network` / `LLMRequiredError`
- triggers 配置：含/不含 各类
- enabled / chain 边界

**用例**：

| ID | 输入 | 期望 | 优先级 |
|---|---|---|---|
| **API-FALLBACK-401** | chain=[claude]，executor 直接 return | `usedStep===chain[0]`，`hops==[]`（happy path 零开销） | 💀 P0 |
| **API-FALLBACK-402** | chain=[claude,deepseek]，claude 抛 429-quota（in triggers），deepseek success | usedStep===chain[1]，hops.length===1，hops[0].errorClass==='429-quota' | 💀 P0 |

| **API-FALLBACK-405** | chain=[claude,deepseek]，claude 抛 429-rate，triggers=`['429-quota','5xx']` | 短路（429-rate 不在 triggers） | 🔴 |
